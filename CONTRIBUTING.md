# Как контрибьютить

Спасибо, что хотите улучшить Awesome AI Handbook!

---

## Что приветствуется

- 🇷🇺 **Русскоязычные ресурсы** (каналы, статьи, курсы, подкасты)
- 📊 **Бенчмарки** на вашем железе (любой чип / GPU)
- 📝 **Черновики туториалов** (Markdown + код)
- 🐛 **Исправления и обновления** (ссылки устарели, версии изменились)

---

## Правила

1. Каждая ссылка — с кратким описанием «зачем это полезно»
2. Проект должен быть активным (коммиты за последние 3–6 месяцев)
3. PR в соответствующий файл раздела (см. структуру ниже)
4. Пометка `last-verified: YYYY-MM-DD` в описании ссылки

---

## Куда добавлять

| Тип контента | Файл |
|--------------|------|
| Новая модель / обновление каталога | `local-models/catalog.md` |
| Рекомендации по задачам | `local-models/models.md` |
| Инструмент инференса | `local-models/tools.md` |
| Бенчмарк Apple Silicon | `local-models/benchmarks/apple-silicon.md` |
| Проблема и решение | `local-models/troubleshooting.md` |
| Фреймворк агентов | `agents/frameworks.md` |
| Туториал агента | `agents/tutorials/` |
| Практический кейс | `use-cases/` |
| RU/EN ресурс | `resources/ru.md` или `resources/en.md` |

---

## Стиль

- Язык: русский для основного контента, английский для `README.md` и `resources/en.md`
- Таблицы предпочтительнее длинных списков
- Команды и код — проверенные, copy-paste ready
- Не дублируйте большие блоки: ссылка на детальный файл лучше копипасты

---

## Процесс

1. Fork → branch → изменения
2. Проверьте, что ссылки рабочие
3. Откройте PR с кратким описанием «что и зачем»

Вопросы — через Issues в репозитории.
