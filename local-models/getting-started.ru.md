## 1. Что такое локальные модели и зачем они нужны

**Локальная AI-модель** — это программа, которая работает прямо на вашем компьютере.
Вы скачиваете её один раз, и она работает полностью офлайн, бесплатно и без ограничений.

**Чем это отличается от ChatGPT или DeepSeek:**

| ChatGPT / Claude | Локальная модель |
|---|---|
| Нужен интернет | Работает офлайн |
| Плата за каждый запрос | Бесплатно (только электричество) |
| Ваши данные уходят на сервер | Всё остаётся на вашем Mac |
| Лимит сообщений | Нет лимитов |
| Нельзя кастомизировать | Можно настроить под себя |

**Что вы сможете делать после этого гайда:**
- Задавать вопросы модели прямо в терминале (как в чате)
- Подключить модель к редактору кода для автодополнения
- Использовать модель для перевода, анализа текста, написания кода
- Всё это — бесплатно и без интернета

## 2. Как открыть терминал (консоль)

Терминал — это окно, где вы вводите команды текстом, а не нажимаете на кнопки.
Многие команды в этом гайде нужно вводить именно в терминале.

### Как найти Terminal на Mac

**Способ 1 — через Spotlight (быстро):**
1. Нажмите `Cmd (⌘) + Пробел` — появится поиск
2. Начните печатать «Terminal» или «Терминал»
3. Нажмите `Enter` — откроется окно терминала

**Способ 2 — через Finder:**
1. Откройте Finder (значок синего лица в Dock)
2. В меню выберите `Переход → Утилиты`
3. Найдите «Терминал» и дважды кликните

### Как выглядит терминал

После открытия вы увидите чёрное (или белое) окно с примерно такой строкой:

```
MacBook-Air:~ user$
```

или

```
user@MacBook-Air ~ %
```

Это называется **приглашение командной строки** (prompt). Оно означает, что терминал готов принять команду.

### Ваша первая команда

Скопируйте эту команду и вставьте в терминал (правой кнопкой мыши → Вставить, или `Cmd+V`):

```bash
echo "Привет, я запускаю AI-модель!"
```

Нажмите `Enter`. Терминал должен ответить:

```
Привет, я запускаю AI-модель!
```

Если вы это видите — терминал работает, можно двигаться дальше.

> **Важно:** Все команды в этом гайде, которые начинаются с `$`, нужно вводить без этого символа.
> `$` просто обозначает приглашение командной строки. То есть если написано `$ brew install ollama`,
> вводите только `brew install ollama`.

## 3. Проверка вашего Mac

Чтобы понять, какие модели вам подходят, нужно узнать две вещи:
- **Чип** (процессор) — Apple Silicon (M1, M2, M3, M4) или Intel
- **Память** (RAM) — сколько гигабайт

### Узнайте свой чип

Скопируйте эту команду в терминал и нажмите Enter:

```bash
uname -m
```

**Результат:**
- `arm64` — у вас Apple Silicon (M1/M2/M3/M4) — отлично, модели будут работать быстро
- `x86_64` — у вас Intel Mac — тоже можно, но скорость будет ниже на больших моделях

### Узнайте, сколько у вас памяти (RAM)

```bash
sysctl hw.memsize | awk '{print "RAM: " $2 / 1073741824 " GB"}'
```

Терминал покажет что-то вроде `RAM: 16 GB` или `RAM: 8 GB`.

### Полная информация о системе

```bash
system_profiler SPHardwareDataType | grep -E "Chip|Memory|Processor"
```

Пример результата:

```
Chip: Apple M1
Memory: 16 GB
```

**Что это значит для выбора модели:**

| У вас | Какие модели подходят |
|-------|----------------------|
| 8 GB RAM | Маленькие модели (до 4B параметров) — например, Qwen 3.5 4B |
| **16 GB RAM** ⭐ | **Средние модели (7B–9B)** — универсальный вариант |
| 24–32 GB RAM | Большие модели (14B–32B) |
| 48+ GB RAM | Очень большие модели (70B+) |

> У большинства MacBook Air / Pro базовой комплектации — **16 GB RAM**. Это оптимальный объём для локальных моделей.

## 4. Установка Homebrew

**Homebrew** — это «магазин приложений» для терминала. Через него удобно устанавливать программы,
которые не найти в App Store.

### Проверьте, установлен ли Homebrew

Некоторые команды могут не сработать, если Homebrew уже есть. Давайте проверим:

```bash
brew --version
```

**Если вы видите** что-то вроде `Homebrew 4.x.x` — Homebrew уже установлен. Поздравляю, пропускайте этот шаг и переходите к [установке Ollama](#5-установка-ollama).

**Если вы видите** `command not found: brew` — Homebrew не установлен, выполняйте следующую команду.

### Установка Homebrew

Скопируйте ЭТУ команду целиком и вставьте в терминал:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

После нажатия Enter:
1. Терминал спросит пароль — введите пароль от вашего Mac (символы не будут отображаться при вводе — это нормально). Нажмите Enter.
2. Homebrew начнёт установку. Это может занять 1–5 минут.
3. Дождитесь сообщения `Installation successful!`

### Проверка установки

```bash
brew --version
```

Должно показать:

```
Homebrew 4.x.x
```

### Если на Apple Silicon появилось предупреждение «not in PATH»

При установке Homebrew на M1/M2/M3/M4 может появиться сообщение:
`Warning: /opt/homebrew/bin is not in your PATH...`

Скопируйте и выполните команду, которую предложит установщик. Обычно это:

```bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zshrc
```

Затем:

```bash
source ~/.zshrc
```

После этого проверьте `brew --version` снова.

## 5. Установка Ollama

**Ollama** — самая простая программа для запуска AI-моделей. Она умеет:
- Сама скачивать модель по вашему запросу
- Запускать её одной командой
- Работать как сервер для других программ

### Установка через Homebrew

```bash
brew install ollama
```

Подождите, пока установка завершится. Это занимает 10–30 секунд.

### Проверка установки

```bash
ollama --version
```

Должно показать: `ollama version 0.x.x`

### Запуск сервера Ollama

После установки Ollama нужно запустить её фоновый сервер:

```bash
ollama serve
```

После этой команды терминал зависнет и покажет что-то вроде:

```
[GIN] 2026/07/24 ... Listening on 127.0.0.1:11434
```

**Это нормально.** Это значит, что сервер работает. Не закрывайте это окно терминала — сервер должен быть запущен всё время, пока вы работаете с моделями.

**Что делать:** откройте **новое** окно терминала (`Cmd+N` или через Spotlight → Terminal) и продолжайте работу в нём. Первый терминал останется с запущенным сервером.

### Проверка, что сервер работает

В новом окне терминала выполните:

```bash
curl http://localhost:11434/api/version
```

Если видите что-то вроде `{"version":"0.x.x"}` — всё работает.

> **Подсказка:** При перезагрузке Mac сервер Ollama нужно запускать снова. Чтобы он запускался автоматически, добавьте приложение «Ollama» в автозагрузку (см. ниже).

### Автоматический запуск (удобно)

После установки в системе появляется приложение **«Ollama»** — найдите его через Spotlight или в папке «Программы».
Запустите его один раз — оно само будет запускать сервер при каждом включении Mac. Иконка Ollama появится в верхней строке меню.

## 6. Запуск первой модели

### Команда для запуска

В терминале (там, где вы **не** запускали `ollama serve`) введите:

```bash
ollama run qwen3.5:4b
```

**Что произойдёт:**
1. Если модель ещё не скачана — Ollama начнёт скачивание (покажет прогресс-бар)
2. Размер модели: ~3.4 GB (как один фильм в HD)
3. Время скачивания: 1–10 минут, зависит от скорости интернета
4. После загрузки — появится приглашение `>>>` — модель готова к диалогу

> **Медленный интернет?** Скачивание происходит один раз. После этого модель работает офлайн.

### Как общаться с моделью

Просто печатайте свои вопросы и нажимайте Enter:

```
>>> Напиши стих про программиста на Python
```

Модель напечатает ответ. После завершения снова появится `>>>` — можно задавать следующий вопрос.

### Полезные команды во время диалога

- `/bye` — выйти из чата (модель выгрузится из памяти)
- `/clear` — очистить историю диалога (начать новый разговор)
- `/show info` — показать информацию о модели
- `Ctrl+C` — прервать генерацию, если модель слишком долго отвечает

### Что попробовать спросить

```
>>> Объясни, что такое рекурсия, простыми словами
>>> Напиши функцию на Python, которая проверяет, является ли число простым
>>> Переведи на английский: "Мне нужно запустить AI-модель локально"
>>> Напиши план изучения Python для новичка
```

### Как запустить другую модель

Выходите из чата (`/bye`) и запускаете другую модель:

```bash
ollama run qwen2.5-coder:7b    # модель для программирования
ollama run llama3.3:8b         # модель от Meta
ollama run phi4-mini           # маленькая, но умная модель
```

**Разница между моделями:**
- `qwen3.5:4b` (3.4 GB) — универсальный чат, быстрый
- `qwen2.5-coder:7b` (4.7 GB) — лучше пишет код, но чуть медленнее
- `llama3.3:8b` (4.9 GB) — хороший английский, рассуждения
- `phi4-mini` (2.5 GB) — маленькая, отлично считает и логически мыслит

> Если вы на MacBook с **16 GB RAM** — все эти модели поместятся.
> Если у вас **8 GB** — запускайте только `qwen3.5:4b` или `phi4-mini`.

### Где искать другие модели

Полный список моделей на Ollama: [ollama.com/search](https://ollama.com/search)
На странице модели указан её размер (колонка Size) — ориентируйтесь на него при выборе.

Чтобы скачать модель без запуска (просто на будущее):

```bash
ollama pull qwen3.5:9b    # скачать, но не запускать
```

## 7. Установка LM Studio (альтернатива без терминала)

Если вам совсем не хочется работать с командной строкой — есть **LM Studio**.
Это программа с обычным интерфейсом (кнопки, меню, поиск), где всё делается мышкой.

### Установка

1. Откройте сайт [lmstudio.ai](https://lmstudio.ai) в браузере
2. Нажмите «Download for macOS» — скачается файл `.dmg`
3. Откройте скачанный файл (обычно в папке «Загрузки»)
4. Перетащите иконку LM Studio в папку «Программы»
5. Откройте LM Studio через Launchpad или Spotlight

### Первый запуск

1. LM Studio предложит найти модель — в поиске введите «Qwen 3.5 4B»
2. Выберите модель из списка, нажмите «Download»
3. Дождитесь скачивания (показывает прогресс)
4. Нажмите «Load» — модель загрузится в память
5. Нажмите «Chat» — откроется окно чата

Всё готово. Дальше просто печатаете вопросы — как в ChatGPT, но локально.

### Когда что выбирать

| Если вы… | Используйте |
|----------|------------|
| Хотите просто чат, как ChatGPT | LM Studio |
| Планируете подключать модели к программам | Ollama |
| Хотите и то, и другое | Ollama (сервер) + Open WebUI (веб-чат сверху) |
