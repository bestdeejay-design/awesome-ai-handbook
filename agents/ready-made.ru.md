## 1. Как читать эту таблицу

Все агенты разделены на **5 категорий** по способу использования:

| Категория | Примеры | Для кого |
|-----------|---------|----------|
| **CLI-агенты** | Aider, Codex CLI, Claude Code | Те, кто работает в терминале |
| **IDE-плагины** | Continue, Cline, Cody | Те, кто работает в VS Code / JetBrains |
| **AI-native IDE** | Cursor, Windsurf | Те, кто хочет IDE с AI «из коробки» |
| **Автономные агенты** | OpenHands, AutoCodeRover | Те, кто хочет «поставил задачу — получил результат» |
| **Агенты-оркестраторы** | Sisyphus, CrewAI, AutoGen, LangGraph, Dify | Те, кто строит мульти-агентные команды «из коробки» |

**Уровни интеграции с локальными моделями:**
- 🟢 **Полная** — агент работает с любой моделью через Ollama / OpenAI-совместимый API
- 🟡 **Ограниченная** — локальные модели подключаются, но не все функции доступны
- 🔴 **Только облако** — агент требует проприетарную модель

## 3. CLI-агенты

Работают в терминале, правят файлы, работают с git. Идеально для тех, кто не хочет выходить из консоли.

### Aider 🥇

[GitHub →](https://github.com/paul-gauthier/aider) · ⭐ 25K+ · 🟢 Полная поддержка локальных моделей

**Что делает:** Редактирует код в git-репозитории по текстовым командам. Сам вносит правки, запускает тесты, создаёт коммиты.

**Ключевая фишка:** Работает в паре моделей — одна архитектор (планирует), вторая редактор (пишет код). Поддерживает «ленивый» режим для дешёвых моделей.

```bash
pip install aider-install
aider-install

# Запуск с локальной моделью
aider --model ollama/qwen2.5-coder:7b

# Режим архитектора (2 модели)
aider --model ollama/qwen2.5-coder:7b --editor-model ollama/qwen3:4b
```

**Когда брать:** Повседневная разработка в терминале. Рефакторинг. Многофайловые изменения.

### Codex CLI

[GitHub →](https://github.com/openai/codex) · ⭐ 15K+ · 🔴 Только облако (OpenAI)

**Что делает:** Официальный CLI-агент от OpenAI. Запускает код в sandbox-окружении, правит файлы.

**Ключевая фишка:** Нативная интеграция с OpenAI API, sandbox-изоляция, поддержка расширений.

```bash
npm install -g @openai/codex
codex "добавь сортировку в список задач"
```

### Claude Code

[GitHub →](https://github.com/anthropics/claude-code) · ⭐ 10K+ · 🔴 Только облако (Anthropic)

**Что делает:** CLI-агент от Anthropic. Работает с Claude.

**Ключевая фишка:** Сильная безопасность, поддержка CLAUDE.md для инструкций, работа с большим контекстом.

### Codebuff

[GitHub →](https://github.com/richardschneider/codebuff) · ⭐ 1K+ · 🟢 Полная поддержка локальных моделей

**Что делает:** Лёгкий CLI-агент для файловых операций.

**Ключевая фишка:** Простота. Одна команда — одно изменение. Никаких sandbox'ов.

## 4. IDE-плагины

Расширения для VS Code, JetBrains и других редакторов. Добавляют AI-функции прямо в интерфейс.

### Continue 🥇

[GitHub →](https://github.com/continuedev/continue) · ⭐ 25K+ · 🟢 Полная поддержка локальных моделей

**Что делает:** Бесплатный Copilot с локальной моделью. Tab autocomplete, чат, инлайн-редактирование, @codebase.

**Ключевая фишка:** Единственный IDE-плагин с полной поддержкой Ollama + MCP-серверов + кастомных правил. Настраивается через JSON.

```json
{
  "models": [{
    "title": "Local Coder",
    "provider": "ollama",
    "model": "qwen2.5-coder:7b",
    "apiBase": "http://localhost:11434"
  }],
  "tabAutocompleteModel": {
    "provider": "ollama",
    "model": "qwen2.5-coder:1.5b"
  }
}
```

**Когда брать:** Если вы в VS Code и хотите бесплатный аналог Copilot с локальной моделью.

### Cline

[GitHub →](https://github.com/cline/cline) · ⭐ 15K+ · 🟢 Полная поддержка локальных моделей

**Что делает:** Автономный агент внутри VS Code. Сам читает файлы, пишет код, запускает команды.

**Ключевая фишка:** Умеет сам создавать и выполнять планы. Поддерживает MCP-серверы.

**Когда брать:** Если нужен автономный агент внутри редактора, а не просто автодополнение.

### Cody

[GitHub →](https://github.com/sourcegraph/cody) · ⭐ 5K+ · 🟡 Ограниченная поддержка локальных моделей

**Что делает:** AI-ассистент от Sourcegraph. Чат, автодополнение, команды.

**Ключевая фишка:** Интеграция с Sourcegraph для поиска по коду. Сильная контекстная осведомлённость.

## 5. AI-native IDE

Полноценные IDE, где AI встроен в каждый уголок интерфейса.

### Cursor

[Сайт →](https://cursor.sh/) · 🟡 Ограниченная поддержка локальных моделей

**Что делает:** Форк VS Code со встроенным AI на каждом уровне. Tab, чат, агент, терминал.

**Ключевая фишка:** Самый отполированный AI-опыт. Agent mode сам планирует и выполняет multi-step задачи.

> **С локальными моделями:** Cursor умеет подключать кастомные модели через OpenAI-совместимый API. Не все функции работают — часть завязана на проприетарные модели Cursor.

### Windsurf

[Сайт →](https://codeium.com/windsurf) · 🟡 Ограниченная поддержка локальных моделей

**Что делает:** IDE от Codeium. Cascade — режим, где AI видит весь проект.

**Ключевая фишка:** «Cascade» — агент, который понимает весь код и может автономно выполнять задачи.

## 6. Автономные агенты

Запускаются в изолированном окружении и выполняют задачи от начала до конца без участия человека.

### OpenHands 🥇

[GitHub →](https://github.com/All-Hands-AI/OpenHands) · ⭐ 40K+ · 🟢 Полная поддержка локальных моделей

**Что делает:** Автономный software engineer в Docker-контейнере. Создаёт план → пишет код → запускает → чинит баги → делает PR.

**Ключевая фишка:** Полная изоляция (Docker), поддержка веб-браузера, работа с локальными моделями.

```bash
docker run -it --rm \
  -e LLM_MODEL=qwen2.5-coder:7b \
  -e LLM_API_KEY=ollama \
  -e LLM_BASE_URL=http://host.docker.internal:11434 \
  -v $(pwd)/workspace:/workspace \
  ghcr.io/all-hands-ai/openhands:latest
```

**Когда брать:** Если нужно «запустил и забыл». Автоматическое выполнение задач, багфикс, создание PR.

### AutoCodeRover

[GitHub →](https://github.com/nus-apr/auto-code-rover) · ⭐ 3K+ · 🟢 Полная поддержка локальных моделей

**Что делает:** Автономный агент для исправления багов и реализации фич.

**Ключевая фишка:** Сфокусирован на ремонте кода. Умеет исследовать код, локализовать баг, предложить фикс.

## 7. Агенты-оркестраторы

Управляют командой специализированных подагентов — сами не пишут код, но координируют тех, кто пишет.

### Sisyphus (OhMyOpenAgent)

[GitHub →](https://github.com/code-yeongyu/oh-my-openagent) · ⭐ 66K+ · 🟢 Полная поддержка локальных моделей

**Что делает:** Плагин для OpenCode / Codex CLI, добавляющий мульти-агентную архитектуру. **11 агентов**, 54+ lifecycle-хука, 5 встроенных MCP, Team Mode. Существует в двух редакциях:

| Редакция | Для | Установка | Агенты |
|----------|-----|-----------|--------|
| **Ultimate** | OpenCode | `bunx oh-my-openagent install` | Все 11 |
| **Light** | Codex CLI | `npx lazycodex-ai install` | 8 компонентов (без оркестрации) |

**Ключевая фишка:** Единственная система, где каждый агент использует свою модель, оптимальную под задачу, а оркестратор (Atlas) управляет их работой как дирижёр оркестром.

#### Команда агентов

| Агент | Роль | Модель по умолчанию |
|-------|------|-------------------|
| **Sisyphus** (я) | Оркестратор верхнего уровня | `claude-opus-4-8` / `kimi-k3` |
| **Hephaestus** | Глубокий исполнитель | `gpt-5.6-sol` |
| **Prometheus** | Стратегический планировщик | `claude-opus-4-8` / `gpt-5.6-sol` |
| **Atlas** | Дирижёр выполнения планов | `claude-sonnet-4-6` / `kimi-k3` |
| **Metis** | Анализ требований, поиск скрытых intent'ов | `claude-sonnet-4-6` |
| **Momus** | QA, ревью планов | `gpt-5.6-terra` |
| **Oracle** | Архитектура, отладка | `gpt-5.6-sol` |
| **Sisyphus-Junior** | Исполнитель задач | `claude-sonnet-4-6` / `kimi-k3` |
| **Explore** | Поиск по коду (grep) | `gpt-5.4-mini-fast` |
| **Librarian** | Документация, OSS | `gpt-5.4-mini-fast` |
| **Multimodal-Looker** | Анализ изображений, PDF | — |

**Как работает (реальная архитектура из репозитория):**

```
Пользователь
    │
    ▼
┌─────────────────────────────────────────────────────┐
│  Planning Layer (Prometheus + Metis + Momus + Oracle)│
│  ┌──────────┐   ┌──────────┐   ┌──────┐  ┌──────┐  │
│  │Prometheus├──►│  Metis   │   │Momus │  │Oracle│  │
│  │(планирует)│   │(анализир.)│   │(ревью)│  │(ревью)│  │
│  └────┬─────┘   └──────────┘   └──────┘  └──────┘  │
│       │          ▲               ▲         ▲        │
│       └──────────┴───────┬───────┘         │        │
│                          │                 │        │
│                     ┌────┴─────┐            │        │
│                     │ .omo/plans/*.md       │        │
│                     └──────────┘            │        │
└─────────────────────────────────────────────┼────────┘
                                              │
    /start-work                               │
    ▼                                         │
┌─────────────────────────────────────────────┼────────┐
│  Execution Layer (Atlas)                    │        │
│  ┌────────────────────────────────────────┐ │        │
│  │  Atlas (conductor) читает план         │ │        │
│  │  и делегирует задачи:                  │ │        │
│  │                                        │ │        │
│  │  task(category="deep") ───► Junior     │ │        │
│  │  task(subagent_type="oracle") ──► Oracle │        │
│  │  task(category="visual-engineering")   │ │        │
│  │      ───► Frontend                     │ │        │
│  │  explore ───► Explore agent            │ │        │
│  └────────────────────────────────────────┘ │        │
└─────────────────────────────────────────────┼────────┘
                                              │
    Результат ◄───────────────────────────────┘
```

**Когда брать:** Если хотите готовую мульти-агентную команду в редакторе — 11 агентов с разными ролями и моделями, которые не нужно настраивать вручную.

> **Этот хендбук написан этой же командой.**

#### Как установить

```bash
# Ultimate Edition (OpenCode)
bunx oh-my-openagent install

# Light Edition (Codex CLI)
npx lazycodex-ai install

# Обе версии сразу
bunx oh-my-openagent install --platform=both
```

## 8. Какой агент выбрать

| Ваш сценарий | Рекомендация |
|-------------|-------------|
| **«Хочу автодополнение в VS Code + чат с локальной моделью»** | [Continue](#continue-) |
| **«Рефакторю код в терминале, нужен git-интеграция»** | [Aider](#aider-) |
