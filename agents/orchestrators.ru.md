> Оркестратор с субагентами — это система, где один «главный» агент управляет командой специализированных агентов, каждый со своей моделью и ролью.  
> Это не фреймворк (LangGraph, CrewAI), а **готовая конфигурация**, которая работает «из коробки».

## 1. Что такое оркестратор с субагентами

**Обычный AI-агент:** одна модель → одна задача → один ответ.

**Оркестратор с субагентами:** одна задача → планировщик → несколько специализированных агентов с разными моделями → синтез результата.

```
Задача
  │
  ▼
┌──────────────┐
│  Оркестратор │  ← решает, кто что делает
└──────┬───────┘
       │
  ┌────┼────┬────┬────┐
  ▼    ▼    ▼    ▼    ▼
 A1   A2   A3   A4   A5   ← разные агенты, разные модели
 │    │    │    │    │
 └────┴────┴────┴────┘
       │
       ▼
    Результат
```

**Ключевой принцип:** каждый агент использует модель, оптимальную под свою роль. Дешёвые модели — для grep и поиска, дорогие — для архитектуры и планирования.

На момент написания (июль 2026) **единственная** система с такой архитектурой «из коробки» — OhMyOpenAgent.

## 2. OhMyOpenAgent (Sisyphus)

[GitHub →](https://github.com/code-yeongyu/oh-my-openagent) · ⭐ 66K+ · TypeScript

**Автор:** [@code-yeongyu](https://github.com/code-yeongyu)  
**Компания:** [Sisyphus Labs](https://sisyphuslabs.ai)  
**Лицензия:** SUL-1.0  
**Релизов:** 222+ · **Контрибьюторов:** 300

### Архитектура (11 агентов)

OhMyOpenAgent содержит **11 встроенных агентов** в трёх слоях:

#### Слой планирования (Planning Layer)
| Агент | Роль | Модели |
|-------|------|--------|
| **Prometheus** | Стратегический планировщик. Интервью, исследование, создание плана | `claude-opus-4-8`, `gpt-5.6-sol`, `glm-5.2` |
| **Metis** | Анализ требований. Ищет скрытые intent'ы, неоднозначности, AI-slop | `claude-sonnet-4-6`, `claude-opus-4-8` |
| **Momus** | QA-ревью плана. Проверяет полноту, выполнимость, непротиворечивость | `gpt-5.6-terra`, `gpt-5.6-sol` |

#### Слой исполнения (Execution Layer)
| Агент | Роль | Модели |
|-------|------|--------|
| **Atlas** 🏆 | Дирижёр выполнения. Читает план, делегирует задачи субагентам, аккумулирует знания | `claude-sonnet-4-6`, `kimi-k3` |
| **Sisyphus** | Главный оркестратор. `ultrawork` — вход в режим полной автономной работы | `claude-opus-4-8`, `kimi-k3` |

#### Слой исполнителей (Worker Layer)
| Агент | Роль | Модели |
|-------|------|--------|
| **Hephaestus** 🏆 | Глубокий исполнитель. Получает цель → сам исследует, планирует, пишет код | `gpt-5.6-sol` |
| **Sisyphus-Junior** | Фокусированный исполнитель. Получает детальный промпт → выполняет | `claude-sonnet-4-6`, `kimi-k3` |
| **Oracle** | Архитектура и отладка. Read-only консультант | `gpt-5.6-sol`, `gemini-3.1-pro` |
| **Explore** | Исследователь кода. Contextual grep | `gpt-5.4-mini-fast` |
| **Librarian** | Документация и OSS. Поиск внешних референсов | `gpt-5.4-mini-fast` |
| **Multimodal-Looker** | Анализ изображений, PDF, медиа | — |

### Как это выглядит на практике

**Простой путь:** написать `ulw` или `ultrawork` в чате → Sisyphus сам решает, какие агенты нужны, и выполняет задачу.

**Точный путь:** `@plan "сделать X"` → Prometheus интервьюирует, создаёт план в `.omo/plans/` → Momus + Oracle ревьюят → `/start-work` → Atlas читает план и делегирует задачи субагентам.

**Глубокий путь:** переключиться на Hephaestus → Hephaestus сам исследует, планирует и выполняет как автономный deep worker.

### Режимы работы

| Режим | Команда | Что делает |
|-------|---------|-----------|
| `ultrawork` / `ulw` | `ulw сделай X` | Полная автономная работа, не останавливается пока не готово |
| `@plan` | `@plan сделай X` | Prometheus планирует, Atlas исполняет |
| `/start-work` | После плана | Atlas выполняет план из `.omo/plans/` |
| Team Mode | `team_create(...)` | До 8 параллельных членов команды |
| `hyperplan` | `hyperplan X` | 5 враждебно настроенных критиков планируют |
| `search` | `search X` | Поиск по коду и документации |
| `analyze` | `analyze X` | Анализ кода |

### Две редакции

| | Ultimate (OpenCode) | Light (Codex CLI) |
|---|---|---|
| **Агенты** | 11 агентов с оркестрацией | 8 компонентов, без оркестрации |
| **Модели** | Своя модель на каждого агента | Модель Codex CLI |
| **Team Mode** | ✅ До 8 участников | ❌ |
| **Lifecycle-хуки** | 54+ | ✅ (ограниченно) |
| **MCP** | 5 встроенных + свои | Plugin-scoped MCP |
| **Установка** | `bunx oh-my-openagent install` | `npx lazycodex-ai install` |
| **Требования** | Bun + OpenCode | Node.js + Codex CLI |

### История названий

Проект несколько раз переименовывался:
1. **oh-my-opencode** — оригинальное название (пакет на npm до сих пор так называется)
2. **oh-my-openagent** — текущее название (с March 2026)
3. **lazycodex** — версия для Codex CLI
4. **omo** — внутреннее сокращение (не путать с npm-пакетом `omo`, это другой проект!)

## 3. Другие системы, приближающиеся к этому

Полных аналогов OhMyOpenAgent нет, но некоторые системы имеют отдельные элементы мульти-агентности.

### Claude Code (sub-agents)

[GitHub →](https://github.com/anthropics/claude-code) · ⭐ 139K+

**Что есть:** Claude Code поддерживает sub-agents — может создавать дочерних агентов для параллельного выполнения задач. Есть `CLAUDE.md` для инструкций, MCP-серверы.

**Чего нет:** Нет разделения на роли (архитектор vs исполнитель vs исследователь). Одна модель на всё. Нет планировщика, отдельного от исполнителя.

### Cline — Kanban mode

[GitHub →](https://github.com/cline/cline) · ⭐ 65K+

**Что есть:** Kanban-доска (`npx kanban`) для параллельного запуска нескольких агентов. Каждый агент работает над своей задачей независимо. Поддержка MCP-серверов.

**Чего нет:** Нет оркестратора, который бы распределял задачи. Нет разных моделей на разные роли. Нет планировщика.

### Aider — Architect/Editor mode

[GitHub →](https://github.com/paul-gauthier/aider) · ⭐ 48K+

**Что есть:** Режим двух моделей: одна (architect) планирует изменения, вторая (editor) пишет код. Поддержка Ollama.

**Чего нет:** Всего два агента, без ролевой специализации. Нет субагентов для исследования/документации.

### OpenHands — Agent Canvas

[GitHub →](https://github.com/All-Hands-AI/OpenHands) · ⭐ 82K+

**Что есть:** Web UI для запуска нескольких агентов. ACP-протокол для координации. Поддержка разных моделей.

**Чего нет:** Каждый агент — это копия одного и того же пайплайна, нет ролевого разделения. Нет планировщика.

## 5. Установка OhMyOpenAgent

### Способ 1: Ultimate Edition (OpenCode) — рекомендуется

```bash
# Установка через Bun
bunx oh-my-openagent install
```

Интерактивный TUI (Terminal UI) проведёт через:
1. Выбор подписки (ChatGPT, Kimi Code, GLM Coding Plan)
2. Настройку провайдеров (Anthropic, Gemini, Copilot, Z.ai)
3. Распределение моделей по 11 агентам
4. Настройку Team Mode

**Рекомендуемые подписки:**
- ChatGPT ($20/мес) — для GPT-5.6 Sol
- Kimi Code ($19/мес) — для Kimi K3 (отличная альтернатива Claude)
- GLM Coding Plan ($10/мес) — для GLM-5

### Способ 2: Light Edition (Codex CLI)

```bash
# Установка через npm (Node.js, не требует Bun)
npx lazycodex-ai install

# Режим без TUI с автономными разрешениями
npx lazycodex-ai install --no-tui --codex-autonomous
```

Установит 8 компонентов в `~/.codex/plugins/cache/sisyphuslabs/omo/`.

### Способ 3: Обе редакции сразу

```bash
bunx oh-my-openagent install --platform=both
```

### Установка через ИИ-агента (рекомендуется авторами)

Скопируйте этот промпт в Claude Code, Cursor, AmpCode или любой другой агент:

```
Install and configure oh-my-openagent by following the instructions here:
https://raw.githubusercontent.com/code-yeongyu/oh-my-openagent/refs/heads/dev/docs/guide/installation.md
```

Агент прочитает полный гайд, выполнит каждый шаг и настроит всё автоматически.

### Важные замечания

1. **Имя пакета:** npm-пакет всё ещё называется `oh-my-opencode` (dual-published как `oh-my-openagent`). Команда `bunx oh-my-openagent install` работает.
2. **Не путайте `omo`:** пакет использует `omo` как bin alias, но НЕ используйте `npx omo` — это другой npm-пакет.
3. **OpenCode vs Codex CLI:** Ultimate-редакция требует OpenCode. Light-редакция работает с официальным Codex CLI от OpenAI.
4. **Telemetry:** анонимная телеметрия включена по умолчанию (1 запрос/день). Отключить: `"telemetry": false` в конфиге.

### После установки

```bash
# Проверка статуса
bunx oh-my-openagent doctor

# Базовая команда — просто напишите:
ulw <ваша задача>

# Или точный путь:
@plan <описание задачи>   # Prometheus планирует
/start-work               # Atlas выполняет
```
