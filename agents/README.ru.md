## Что такое AI-агент

**Обычный чат:** вы задали вопрос → модель ответила. И всё.

**AI-агент:** модель сама решает, что делать, вызывает инструменты, анализирует результат и повторяет, пока не достигнет цели.

Простой пример — вместо того чтобы спрашивать «Какая погода в Токио?» и получать ответ «Я не знаю, у меня нет доступа к интернету», агент сам вызывает API погоды, получает данные и формулирует ответ.

```
Чат:  Вопрос → Ответ
Агент: Вопрос → Агент решает вызвать инструмент → Получает данные → Формулирует ответ
```

### Из чего состоит агент

```
┌─────────────────────────────────────────────────┐
│                  AI-агент                        │
│                                                  │
│  ┌──────────┐  ┌──────────┐  ┌───────────────┐  │
│  │   LLM    │  │ Инструменты│  │ Цикл принятия  │  │
│  │ (модель) │  │ (tools)   │  │   решений      │  │
│  └──────────┘  └──────────┘  └───────────────┘  │
│       │              │               │           │
│       ▼              ▼               ▼           │
│  Принимает     Поиск в web,    Анализ → Действие  │
│  решения       API, код,       → Наблюдение →     │
│                файлы            Повтор            │
└─────────────────────────────────────────────────┘
```

**Ключевые компоненты:**

1. **LLM** — языковая модель, которая принимает решения (локально через Ollama)
2. **Инструменты (tools)** — функции, которые модель может вызывать: поиск, калькулятор, работа с файлами, API
3. **Цикл «думай → действуй → наблюдай»** — агент повторяет этот цикл, пока задача не решена
4. **Память** — контекст между шагами (что уже сделано, что ещё нужно)

## Минимальный агент на Python + Ollama

Вот настоящий рабочий агент, который умеет складывать числа (модель сама решает, когда вызвать калькулятор):

```python
import requests
import json

# 1. Определяем инструмент, который будет использовать агент
tools = [
    {
        "type": "function",
        "function": {
            "name": "calculator",
            "description": "Складывает два числа",
            "parameters": {
                "type": "object",
                "properties": {
                    "a": {"type": "number"},
                    "b": {"type": "number"}
                },
                "required": ["a", "b"]
            }
        }
    }
]

# 2. Реальная функция, которую вызовет модель
def calculator(a, b):
    return {"result": a + b}

# 3. Отправляем запрос модели — модель решает, нужен ли инструмент
response = requests.post("http://localhost:11434/api/chat", json={
    "model": "qwen3.5:4b",
    "messages": [
        {"role": "user", "content": "Сколько будет 5 + 3 + 7?"}
    ],
    "tools": tools,
    "stream": False
})

data = response.json()

# 4. Проверяем, хочет ли модель вызвать инструмент
if data["message"].get("tool_calls"):
    for tool_call in data["message"]["tool_calls"]:
        if tool_call["function"]["name"] == "calculator":
            args = tool_call["function"]["arguments"]
            result = calculator(args["a"], args["b"])
            print(f"Модель вызвала калькулятор: {args['a']} + {args['b']} = {result['result']}")
else:
    print("Ответ модели:", data["message"]["content"])
```

Этот код — буквально всё, что нужно для первого агента. Модель сама понимает, что `5 + 3 + 7` можно разбить на два действия, и вызывает калькулятор.

## Чем агент сложнее простого чата

| Простой чат | AI-агент |
|---|---|
| Один ответ | Многошаговый цикл |
| Нет инструментов | Может вызывать функции, API, поиск |
| Не помнит между шагами | Хранит историю действий |
| Ответ всегда текстом | Может возвращать структурированные данные |
| Не проверяет себя | Может анализировать и исправлять ошибки |

**Когда нужен агент:**
- Нужно выполнить последовательность действий (найти → проанализировать → ответить)
- Нужен доступ к внешним данным (поиск, БД, API, файлы)
- Задача требует планирования и разбивки на шаги
- Нужна самопроверка и исправление ошибок

**Когда агент не нужен:**
- Простой вопрос-ответ (обычный чат справится)
- Задача решается одной командой
